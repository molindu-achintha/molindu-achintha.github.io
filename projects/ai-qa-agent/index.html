Crowdbox AI Quality Assurance Agent

Enterprise Multi-Agent System for Automated SDR Call Evaluation with LangGraph Orchestration, RAG-Powered Policy Enforcement, and Salesforce/Zoom Integration

Executive Summary

Crowdbox's AI Quality Assurance Agent automates the evaluation of SDR and cold-calling performance across multiple enterprise campaigns. The system integrates Salesforce Enterprise, Zoom Workspaces, and AI-powered evaluation layers to deliver consistent, objective, and scalable QA scoring. This system replaces manual QA, enabling 100% call coverage, faster feedback loops, and consistent scoring across campaigns like Dell, Check Point, Cisco, Palo Alto, Lenovo, Nutanix, Juniper, Sage, and Schneider Electric.

System Workflow

Upon completion of a call logged in Salesforce, the QA Agent automatically:

  1. Fetches the associated Zoom recording
  2. Transcribes it using Whisper/AssemblyAI with speaker diarization
  3. Analyzes the transcript via a multi-agent AI evaluation framework
  4. Scores the call against campaign-specific rubrics
  5. Writes results and coaching notes back to Salesforce
  6. Updates dashboards and alerts QA managers for anomalies

Lead management, call logging, and result storage. Triggers webhook on "Passed to AM" or "Callback Booked" status.

Call recordings API. Downloads audio files using Meeting ID for transcription.

Webhook Workflow

  1. When a sales call is logged in Salesforce (lead marked as "Passed to AM" or "Callback Booked"), a webhook event is triggered
  2. The webhook sends metadata: Call ID, Zoom Meeting ID, Caller (SDR), Campaign type (Dell, Cisco, Palo Alto, etc.)
  3. The Zoom API is called using the Meeting ID to download the call recording

Outcome

A new call record + its audio recording enters the Ingestion Pipeline for transcription.

4Ô∏è‚É£ Policy / Criteria Store (RAG)

Technology: Pinecone Vector Database

RAG Store Purpose

Retrieval-Augmented Generation store for rubrics and policies. Stores QA rubrics, scoring criteria, campaign scripts, and compliance documentation as embeddings.

How It Works

  • Stores QA rubrics, scoring criteria, campaign scripts as embeddings
  • During evaluation, orchestrator retrieves relevant rubric context (e.g., "Dell campaign QA policy")
  • Injects context into the prompt for LLM agents
  • Enables dynamic context-injection for consistent evaluations across campaigns

End-to-End Process Flow

1 Salesforce/Zoom webhook triggers ingestion
2 Audio downloaded ‚Üí transcribed by AssemblyAI
3 Transcript passed to multi-agent evaluator
4 Worker agents assess each criterion
5 Supervisor + Policy agents validate
6 Output stored in DB ‚Üí surfaced in dashboards
7 Human feedback fine-tunes agents

Salesforce Write-back

The system updates the Lead/Call record with:

  • QA_Score__c - Overall quality score
  • QA_Result__c - Pass/Fail status
  • QA_Coaching_Notes__c - AI-generated coaching recommendations
  • QA_Red_Flags__c - Compliance issues detected
  • Transcript link for reference
